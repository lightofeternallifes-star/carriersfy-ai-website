# Client Onboarding Template — Carriersfy AI

> Standard checklist for every new signed client. Owner: Sofia (when deployed) / Juan. Last updated: 2026-06-25

---

## Pre-Kickoff (Days 1–2)

- [ ] Service agreement signed and filed
- [ ] Invoice sent for setup fee
- [ ] Create client folder in `/OMEGA/CLIENTS/[ClientName]/` with all 9 standard files
- [ ] Create project folder in `/OMEGA/PROJECTS/[ClientName]/` with all 8 standard files
- [ ] Add client to [CLIENT_REGISTRY](../../CORE/CLIENT_REGISTRY.md)
- [ ] Add project to [PROJECT_REGISTRY](../../CORE/PROJECT_REGISTRY.md)
- [ ] Add tasks to [TASK_ENGINE](../../CORE/TASK_ENGINE.md)
- [ ] Send welcome email to client (introduce Juan + Sofia)
- [ ] Schedule kickoff call (45–60 min)
- [ ] Assign AI employees to this client in [AGENT_REGISTRY](../../CORE/AGENT_REGISTRY.md)

---

## Kickoff Call (Week 1)

- [ ] Confirm business overview and pain points
- [ ] Map all communication channels (email, WhatsApp, phone, web)
- [ ] Inventory existing tools (CRM, ERP, scheduling system)
- [ ] Define AI employee scope — what it handles, what it escalates
- [ ] Define escalation paths — when does a human step in?
- [ ] Collect training materials (FAQs, existing email templates, product sheets)
- [ ] Set client expectations: response times, review process, feedback loop
- [ ] Define success metrics — what does a win look like in 90 days?
- [ ] Schedule Week 2 check-in

---

## AI Employee Configuration (Week 1–2)

- [ ] Build knowledge base from training materials
- [ ] Configure AI employee personality and tone (match client's brand voice)
- [ ] Set language (EN / PT-BR / ES)
- [ ] Define conversation flows for top 5 use cases
- [ ] Configure escalation triggers
- [ ] Set up integrations (CRM, WhatsApp, email, etc.)
- [ ] Internal testing — 50+ test conversations
- [ ] Document all configuration in [Current_Build](../../CLIENTS/[ClientName]/Current_Build.md)

---

## Soft Launch (Week 3)

- [ ] Client reviews AI employee in test environment
- [ ] Collect feedback — adjust tone, responses, flows
- [ ] Confirm all integrations are live
- [ ] Final testing with real-world scenarios
- [ ] Client signs off on soft launch

---

## Go Live (Week 4)

- [ ] AI employee is live on client's channel(s)
- [ ] Monitor first 48 hours closely — review all conversations
- [ ] Address any edge cases or mishandled queries
- [ ] Send go-live notification email to client
- [ ] Set up performance reporting cadence (weekly or biweekly)
- [ ] Update [CURRENT_STATUS](../../PROJECTS/[ClientName]/CURRENT_STATUS.md) to [LIVE]
- [ ] Add V1.0 entry to [CHANGELOG](../../PROJECTS/[ClientName]/CHANGELOG.md)

---

## 30-Day Review

- [ ] Compile performance data (conversations handled, escalations, conversion impact)
- [ ] Schedule 30-day review call with client
- [ ] Identify optimization opportunities
- [ ] Document findings in [Meetings](../../CLIENTS/[ClientName]/Meetings.md)
- [ ] Confirm client satisfaction and retainer renewal
